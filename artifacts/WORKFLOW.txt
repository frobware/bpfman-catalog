bpfman Operator Catalog - Workflow Guide
==========================================

Multi-bundle catalog generated successfully with 3 bundles!

Generated Files:
  - fbc-template.yaml   (FBC template with 3 bundles)
  - catalog.yaml        (Rendered catalog, ready to build)
  - Dockerfile.catalog  (Dockerfile for catalog image)
  - Makefile            (Makefile for automated build/push/deploy)


To Deploy Your Catalog:

Step-by-step workflow:
  1. Build:  $ make -C ./artifacts build-catalog-image
  2. Push:   $ make -C ./artifacts push-catalog-image
  3. Deploy: $ make -C ./artifacts deploy-catalog

Shortcut for catalog only (build + push + deploy):
  $ make -C ./artifacts build-and-deploy-catalog

After deploying, the catalog will be available in the OpenShift Console's
OperatorHub where you can manually install the bpfman operator.

For automatic subscription (will install the operator):
  4. Subscribe: $ make -C ./artifacts subscribe

All-in-one workflow (build + push + deploy + subscribe):
  $ make -C ./artifacts all

Custom Registry Examples:

Ephemeral testing with ttl.sh:
  $ make -C ./artifacts IMAGE=ttl.sh/37ae1a28-55a3-422b-a567-d252204eae59-4907d27b-7051-4c3c-b3a7-aabe9f2336a7:29m all

Persistent storage with quay.io:
  $ make -C ./artifacts IMAGE=quay.io/user/my-catalog all

Available Makefile Targets:
  Run $ make -C ./artifacts help to see all available targets and options.

Cleanup:
  To remove all deployed resources:
  $ make -C ./artifacts undeploy
